Web开发三个基石: javascript、html、css,属性就算完成了基础。

HTML

HTML是一种标记语言，用于标记网页如何展示相关内容。
HTML提供一些标签，用于告诉浏览器如何在当前的用户界面上展示数据。

CSS布局

由外到内为： margin -> border -> padding

在 CSS 里一切 HTML 元素皆为盒子，也就是通常所说的盒模型。 块级元素自动从新的一行开始（比如标题、段落以及 div），行内元素排列在上一个元素后（比如图片以及 span）。 元素默认按照这种方式布局称为文档的普通流，同时 CSS 提供了 position 属性来覆盖它。

当元素的定位设置为 relative 时，它允许你通过 CSS 指定该元素在当前文档流页面下的相对偏移量。 CSS 里控制各个方向偏移量的属性是 left、right、top 和 bottom。 它们代表从原来位置向远离该方向偏移指定的像素、百分比或者 em。

absolute，它的含义是相对于其包含块定位。 和 relative 定位不一样，绝对定位会将元素从当前的文档流里面移除，周围的元素会忽略它。 这样我们就可以用 CSS 的 top、bottom、left、right 属性来调整元素的位置。

绝对定位比较特殊的一点是元素的定位参照于最近的 positioned 祖先元素。 如果它的父元素没有添加定位规则（默认是 position: relative;），浏览器会继续寻找直到默认的 body 标签。

fixed 定位，它是一种特殊的绝对（absolute）定位，将元素相对于浏览器窗口定位。 类似于绝对位置，它与 CSS 偏移属性一起使用，并且也会将元素从当前的文档流里面移除。 其它元素会忽略它的存在，这样也许需要调整其他位置的布局。

但 fixed 和 absolute 的最明显的区别在于，前者定位的元素不会随着屏幕滚动而移动。

如果元素有重叠，可以用 z-index 的表示层级，数值越大，层级越高。

媒体查询

媒体查询是 CSS3 中引入的一项新技术，它可以根据不同的视口大小调整内容的布局。 视口是指浏览器中，用户可见的网页内容。 视口会随访问网站的设备不同而改变。

除了使用 em 或 px 设置文本大小，你还可以用视窗单位来做响应式排版。 视窗单位和百分比都是相对单位，但它们是基于不同的参照物。 视窗单位是相对于设备的视窗尺寸（宽度或高度），百分比是相对于父级元素的大小。

四个不同的视窗单位分别是：

  • vw：如 10vw 的意思是视窗宽度的 10%。
  • vh： 如 3vh 的意思是视窗高度的 3%。
  • vmin： 如 70vmin 的意思是视窗的高度和宽度中较小一个的 70%。
  • vmax： 如 100vmax 的意思是视窗的高度和宽度中较大一个的 100%。

display

每个元素都有一个默认的display值，这与元素的类型有关。对于大多数元素它们的默认值通常是 block & inline 。

block 块级元素

类似<div>这类会直接撑满一行的元素就是 块级元素

常见的还有<p> <header> <footer>等

inline 行内元素

类似<span>这里可以出现在段落中的元素为 行内元素

常见的还有<a>等

inline元素都在一行内显示，直到该行排满，对inline元素设置width,height属性无效。

none 隐藏元素

display属性设置为 none， 会隐藏这个元素，和 visibility: hidden; 不一样的是，none 不会占据空间

inline-block

display 设置为 inline-block 后，该元素仍然拥有 inline 特性，不会一个元素就是一行，同时能设置width,height。

position

relative 相对位置

正常的布局方式是从上到下一份一份拼接起来，而 position 这可以做一些特殊的布局。

relative 可以设置相对布局，这个相对是相对于元素原来的位置，比如某个 <div> 设置例子中的css className="box-relative" 后，这个元素会在他原来的位置上面，向下移动 50px(距离顶部50px)， 向右移动 50px(距离左边50px)

absolute 绝对位置

absolute，它的含义是相对于其包含块定位。 和 relative 定位不一样，绝对定位会将元素从当前的文档流里面移除，周围的元素会忽略它。 这样我们就可以用 CSS 的 top、bottom、left、right 属性来调整元素的位置。

绝对定位比较特殊的一点是元素的定位参照于最近的 positioned 祖先元素。 如果它的父元素没有添加定位规则（默认是 position: relative;），浏览器会继续寻找直到默认的 body 标签。

fixed 定位位置

fixed 定位，它是一种特殊的绝对（absolute）定位，将元素相对于浏览器窗口定位。 类似于绝对位置，它与 CSS 偏移属性一起使用，并且也会将元素从当前的文档流里面移除。 其它元素会忽略它的存在，这样也许需要调整其他位置的布局。

但 fixed 和 absolute 的最明显的区别在于，前者定位的元素不会随着屏幕滚动而移动。

常用于 固钉 效果的元素, 将页面元素钉在可视范围。

float

float(悬浮)，常用于文字环绕图片等效果。

给一个div添加这个样式，然后放到一个 <p></p> 之间，就能看到文字环绕的效果

如果不喜欢文字环绕这个元素，就可以对p元素中使用 clear 属性，如果被环绕的div的float为left，只需要给p元素的 clear 也设为left，div就无法被环绕了，会根据元素生产的先后顺序出现在上方或者下方。

flex (重点)

Flex 是 Flexible Box 的缩写，意为”弹性布局”，用来为盒状模型提供最大的灵活性。

启用 flex 布局: display: flex,在该元素启用 flex 布局后，改元素就自动变为 flex 容器，元素的子元素自动变为 flex item (容器成员)

flex 布局常用属性：

作用于容器的：

  • flex-direction
  • flex-flow
  • justify-content
  • align-items
  • align-content
  • flex-wrap

作用于容器成员的：

  • order
  • align-self

设为 Flex 布局以后，子元素的float、clear和vertical-align属性将失效。

在 flex 布局中需要明白一个重要概念：main 和 cross， mian 是当前布局的主轴，cross 是当前布局的交叉轴(垂直于主轴的轴)

flex-direction 设置主轴方向

flex-direction 主要用于设置主轴的方向，也就是容器子元素的排列方向.

  • colunm 设置子元素为 列（垂直方向） 排序, 从上往下
  • row 设置子元素为 行（水平方向） 排序， 从左到右
  • -reverse 则为反向

flex-wrap 设置换行

  • nowrap: 不换行
  • wrap: 正常换行 第一行在上方
  • wrap-reverse: 反向换行 第一行在下方

flex-flow 同时设置方向和换行

flex-flow === flex-direction + flex-wrap

justify-content 设置主轴对齐方式

定义主轴上的对齐方式。

justify-content: flex-start | flex-end | center | space-between | space-around;

start 、end 和 center 比较好理解。 就是靠起点对齐，靠终点对齐和靠中间对齐。

需要注意的是 space-between 和 space-around。

  • space-between 子元素间距相等，两端没有空隙
  • space-around 子元素间距相等，但是两端有一定的空隙，两端的空隙距离为间距的一半

align-items 设置交叉轴对齐方式

定义交叉轴上的对齐方式

align-items: flex-start | flex-end | center | baseline | stretch;

start 、end 和 center 和 justify-content 中的含义是一样的。

  • baseline: 项目的第一行文字的基线对齐
  • stretch: 如果项目未设置高度或设为auto，将占满整个容器的高度。

align-content 设置多根轴线的对齐方式

当存在多个轴的时候，可以通过 align-content 来设置整体的对齐方式。该属性只在多轴生效，单轴不生效。

align-content: flex-start | flex-end | center | space-between | space-around | stretch;

order 子元素权重

order 用于设置子元素在排序中的顺序，数值越小，排列越靠前，默认为0。

align-self 设置子元素的align-items值

可以设置子元素的align-items值，让其表现出不同的对齐方式

align-self: auto | flex-start | flex-end | center | baseline | stretch;

DOM (Document Object Model——文档对象模型)

什么是DOM

Web的运行，需要 html css javascript 共同支持运转。

而 js 是如何操作 html 和 css 的呢？，是通过 DOM.

js 可以在不改动网页的情况下，对网页进行操作，比如添加、去除 和 响应 某些网页内容。

但是 js 并不能直接去修改 html，而协助 js 去修改 html 的就是 DOM，

DOM 和 html 的关系，就像对同一份文章的不同翻译，他们所表达的内容是完全一致的，一个网页里面的 html 有什么内容，一个网页的 DOM 就有什么内容，就像常用的 MVVM 架构中的 view 和 viewmodel 一样双向绑定，DOM 就是类似 viewmodel 一样的存在，html 上面的所有内容都在 DOM 上面有映射，所以只要修改 DOM 里面的内容， html 也会自动修改对应的内容。这就是 js 通过 DOM 操作 html 的方法。

同样的，如果 html 更新了新内容，DOM 也会更新新内容

html 和 DOM 互相关联绑定则是有浏览器处理完成的，不需要我们主动去映射。DOM 等于是浏览器提供给开发者操作 html 提供的工具。

DOM 操作

DOM 方法 主要是 document 对象的一系列 函数。

常见 document 函数

  • document.getElementById() 根据ID返回唯一的元素。
  • document.getElementsByClassName() 根据 class 值 返回元素集合
  • document.querySelector("#container") 选择器，选择id为container的第一个元素并返回
  • document.querySelectorAll(".head-red") 选择器，选择class为head-red的所有元素并返回
  • document.querySelector("#container").children 选择器，选择id为container的第一个元素的所以子元素

npm

前端开发中，node的一种包管理工具(Node Package Manager),
npm是基于node.js开发的工具，也是Node默认的包管理器

以下内容都是使用的 Mac 平台

npm 内容

npm 由三部分组成：

  • 网站 显示所有的包内容
  • 注册表 一个保存了所以包信息的数据库
  • 命令行工具 协助使用者同npm打交道的工具

使用

  • npm -v 查看版本
  • npm install 安装package文件里面的依赖库
  • npm install xxxx 安装具体的库
  • npm list 查看已安装的库， 加上-g 就是查看全局安装的库
  • npm install xxxx -save 安装具体的库，并写入package文件中的dependencies
  • npm install xxxx -save -dev 安装具体的库，并写入package文件中的devDependencies

package.json

npm init 后，会自动生产一个npm的配置文件 package.json,用于编辑和展示当前通过npm管理的库。

默认结构：

scripts 里面可以创建命令，命令的执行语句其实就是 shell 语句

在安装依赖库后，会产生一个 package-lock.json 文件，用于记录安装库的详细信息

在安装依赖后，我们会发现有时候安装的库为:
"moment": "^2.25.3" 或者 "react": "17.0.0",有些版本数字前面会有 ^ ，有些却没有。
这里的 ^ 是指的获取当前大版本下的最新版本。
比如 "react": "17.0.0" 是获取 版本号为 17.0.0 的版本；

而 "react": "^17.0.0" 则是保持 17 不变，0.0 则获取最新的，
假设当前react的最新版本为 18.0.1 ，而17版本的最新小版本为 17.9.4,
那么"react": "^17.0.0"实际获取的是17.9.4

同时还存在 "react": "~17.0.0"，以~为开头，表示只更新最后一位。

x为需要更新的地方，

  • "react": "17.0.0" 表示 “17.0.0”
  • "react": "^17.0.0" 表示 “17.x.x”
  • "react": "~17.0.0" 表示 “17.0.x”

项目构建

什么是项目构建：

将开发者的源代码，经过编译，语法检查，兼容性处理，代码压缩，生成浏览器可识别的，可稳定运行的代码。

src —> dist

现在前端的各种框架都需构建工具将其构建为浏览器能使用的 html、js、css。

为了方便构建，我们通常会使用构建工具来处理这些内容。但是构建工具通常不是开箱即用的，需要开发者自己将其配置成适合项目的，方便使用的工具。

常见构建工具

  • grunt
  • gulp 简单、小项目可用
  • webpack 常用,复杂但是功能全
  • vite 新型工具

webpack

webpack，一个模块打包器，分析项目结构，找到JavaScript模块以及其它的一些浏览器不能直接运行的拓展语言（Scss，TypeScript等），并将其转换和打包为合适的格式供浏览器使用。

核心：

  • Entry 入口，指定打包的起点
  • OutPut 出口，指定打包后的资源输出的具体位置
  • Loader 用于处理 非 JS文件，webpack本身只能处理js文件
  • Plugins 插件，可以用于扩展各种功能，比如压缩代码，设置环境变量之类的
  • Mode 模式，通常为 development 或者 production

webpack 配置文件

在项目中添加一个 webpack.config.js 文件，在里面编码相关的webpack配置

因为webpack是基于node.js运行的，所以在配置文件中，模块化默认使用 CommonJS
和代码的使用是不一样的，代码通常使用 es6
