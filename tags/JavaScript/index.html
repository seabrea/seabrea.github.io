Web开发知识梳理

Web开发知识梳理

Web计算机网络

HTTP

Http协议是建立在TCP协议基础之上的，当客户端需要从服务器获取数据的时候，会发出一次Http请求。Http会通过TCP建立起一个到服务器的连接通道，当本次请求需要的数据完毕后，Http会立即将TCP连接断开，这个过程是很短的。所以Http连接是一种短连接，是一种无状态的连接(无状态连接 即 每次都是一个新连接，数据完了后就释放相关内容，不保留状态)。

常见HTTP请求方法:

  • GET — 获取资源
  • POST — 传输资源
  • PUT — 更新资源
  • DELETE — 删除资源
  • HEAD — 获取报文首部

GET 和 POST 的区别

GET
  1. 浏览器回退或者刷新是无害的，因为GET请求是幂等的(无论多少次请求都不会改变资源的内容)；
  2. 会被浏览器默认主动缓存；
  3. 请求参数会被保留在浏览器的浏览记录中；
  4. GET的参数没有长度限制，但是因为参数在URL上面，很多浏览器对URL有长度限制，所以对GET的参数长度实际造成了一定的影响，不同浏览器对URL长度的限制不一样，如果超过限制通常会返回414
  5. GET的参数放在URL上面，相对安全性更差；
  6. GET主要是对数据进行查询，通常表现为只读；
POST
  1. 浏览器回退或者刷新是有害的，非幂等的，POST会重新请求导致资源可能被修改；
  2. 不会被浏览器主动缓存，但是可以手动设置；
  3. 请求参数是不会被保留在浏览记录中的；
  4. POST的参数在HTTP Body中，没有参数长度限制；
  5. POST的参数在HTTP Body中，安全性相对Get较好；
  6. POST主要对数据进行增删改，表现为读写；
GET和POST的本质区别

GET 和 POST 方法没有实质区别，只是浏览器对这两者的表现给出了一些区别，区别也只在于 HTTP Method 不一样罢了，实际上两种请求方法都可以在路径哪里写上 ? 后的参数，并且两种方法都可以写上 HTTP Body，只是浏览器遵照 HTTP 标准规定，给两种方法加以区别罢了。

简单的说就是 HTTP 规定有这样两种方法，并且有这些特性。所以浏览器就按照规定添加了两个符合特性的方法，但是在浏览器层的两个方法的本质上是一样的。

HTTP报文

HTTP报文通常有 请求报文 和 响应报文 两种

请求报文

通常由 请求行 请求头 和 请求体 三者组成

响应报文

通常由 状态行 响应头 和 响应体 三者组成
